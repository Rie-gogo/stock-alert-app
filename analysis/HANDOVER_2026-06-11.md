## 2. システム構成

### 技術スタック
- **フロントエンド**: React 19 + Tailwind 4 + shadcn/ui
- **バックエンド**: Express 4 + tRPC 11
- **DB**: MySQL（TiDB）/ Drizzle ORM
- **認証**: Manus OAuth
- **データ取得**: kabu STATION® API（Windows PC上で動作）+ J-Quants API

### データフロー
```
Windows PC
└─ kabu STATION® API（ポート18080）
   └─ kabu_board_relay.py（Python中継スクリプト）
      ├─ WebSocket → 板情報 → POST /api/trpc/trading.pushOrderBook
      └─ 1分足ポーリング（毎分:15秒） → POST /api/trpc/trading.pushCandle
                                              ↓
                                    Manusサーバー（realtimeSimEngine.ts）
                                    └─ シグナル判定 → DB保存（rt_trades）
```

## 8. シグナルロジック（detectSignals）

実装済みシグナル（Phase37〜39）:
1. **GoldenCross/DeadCross**: MA5がMA25を上抜け/下抜け
2. **RSI+BB**: RSI買われすぎ(70以上)+BB上限超え → 売り / RSI売られすぎ(30以下)+BB下限割れ → 買い
3. **VWAPクロス**: 価格がVWAPを上抜け → 買い / 下抜け → 売り
4. **長い上ヒゲ**: 上ヒゲが実体の2倍以上 → 売り
5. **長い下ヒゲ**: 下ヒゲが実体の2倍以上 → 買い
6. **はらみ線**: 前足が当足を包む → 転換シグナル
7. **VWAP反発**: VWAPまで下落後の反発 → 買い
8. **ダブルトップ/ダブルボトム**: 過去50本で2つの山/谷 → 転換シグナル
9. **三尊（H&S）/逆三尊**: 頭と肩パターン → 転換シグナル

## 10. このチャットでのルール・方針

1. **情報の正確性最優先**: 誤情報・推測での回答は禁止。コードやDBを実際に確認してから回答する
2. **変更前にgit logで確認**: 「前回変更した」という主張は必ずgit logで裏付けを取る
3. **設定値の混同禁止**: リアルタイムエンジン（0.7%）とバックテスト（2.0%）は別設定。混同しない
4. **スリッページ率の基準**: 最小呼値（ティック）ベースで設定する。H:0.03%, M:0.06%, L:0.10%（片道）が現実的
5. **発注制限の基準**: kabu STATION API制限は秒間5件。今日の最大集中は0.25件/秒で問題なし
6. **バックテスト対象銘柄の区別**:
   - 137日バックテスト: 太陽誘電・村田・さくら・SUMCO・キオクシア（5銘柄）
   - リアルタイムシミュレーション: TARGET_STOCKS全20銘柄
   - SIMULATION_STOCKS（realSimulation.ts）: 10銘柄（バックテスト用）
7. **相場状況の考慮**: 今日（2026-06-11）は上昇相場。通常日の期待値は50〜70%程度と見積もる
8. **元金の基準**: 1銘柄あたり300万円（20銘柄で最大6,000万円）
9. **分析結果の採用基準**: 複数シナリオがある場合は最も利益が上がるケースを優先的に採用する
10. **板情報の活用**: kabu STATION APIが利用可能になり次第、板情報を売買シグナルの一つとして追加することを検討する
